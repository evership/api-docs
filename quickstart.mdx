---
title: "Quickstart"
description: "Ship your first parcel with the Evership API in six steps."
---

<Note>
  You need an API key to follow this guide. Log in to the [Evership Dashboard](https://evership.app), navigate to **Settings > API Keys**, and generate one. Pass it in the `apiKey` header on every request.
</Note>

The Evership shipping flow has six steps: validate addresses, fetch rates, create a shipment, purchase it, retrieve the label, and track it. This guide walks through each one with real cURL commands you can copy and run.

All requests go to the base URL:

```
https://api.evership.app/v1
```

<Steps>

<Step title="Validate addresses (optional)">

Validate the collection and/or delivery address to confirm it is serviceable and correctly geocoded. This catches typos and unsupported areas early.

```bash
curl -X POST https://api.evership.app/v1/validate-address \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY" \
  -d '{
    "address_1": "14 Highland Close",
    "address_2": "",
    "suburb": "Vredehoek",
    "city": "Cape Town",
    "province": "Western Cape",
    "postal_code": "8001",
    "country": "South Africa"
  }'
```

The response returns the geocoded address with coordinates and a `validation_status` of `fully_matched` or `partial_match`. If it's a partial match, check `partial_match_suggestions` for corrected alternatives.

<Info>
  This step is **optional**. If your addresses are already geocoded or you have latitude/longitude coordinates, you can skip validation and pass the coordinates directly when creating the shipment.
</Info>

</Step>

<Step title="Get shipping rates">

Send the collection address, delivery address, and parcel dimensions to receive available rates from multiple carriers. You can optionally filter by `carrier` or `carrier_service`.

```bash
curl -X POST https://api.evership.app/v1/rates \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY" \
  -d '{
    "collection_address": {
      "address_1": "12 Clyde Rd",
      "suburb": "Green Point",
      "city": "Cape Town",
      "province": "Western Cape",
      "postal_code": "8005",
      "country": "South Africa"
    },
    "delivery_address": {
      "address_1": "14 Highland Close",
      "suburb": "Vredehoek",
      "city": "Cape Town",
      "province": "Western Cape",
      "postal_code": "8001",
      "country": "South Africa"
    },
    "parcels": [
      {
        "dimensions": {
          "length": 30,
          "width": 20,
          "height": 15
        },
        "weight": 2.5
      }
    ],
    "carrier": "",
    "carrier_service": "",
    "return_failed_quotes": false
  }'
```

The response returns rates grouped by carrier, each with a `slug` (e.g. `dpd-onx`, `fw-brown`), service level, estimated delivery date, and price. Note the `slug` of your preferred service — you'll use it as the `carrier_service` when creating the shipment.

</Step>

<Step title="Create a shipment">

Create a shipment with sender and receiver details, parcel dimensions, and your chosen carrier service. Set `purchase_label` to `true` to create and purchase in one step, or `false` to create a draft that you can review before purchasing.

```bash
curl -X POST https://api.evership.app/v1/shipments \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY" \
  -d '{
    "shipment_title": "ORDER-12345",
    "shipment_type": "outbound",
    "sender": {
      "name": "Jane Doe",
      "contact": "0761234567",
      "email": "jane@example.com",
      "collection_address": {
        "address_1": "12 Clyde Rd",
        "suburb": "Green Point",
        "city": "Cape Town",
        "province": "Western Cape",
        "postal_code": "8005",
        "country": "South Africa"
      },
      "pickup_instructions": "Ring the bell at gate"
    },
    "receiver": {
      "name": "John Smith",
      "contact": "0769876543",
      "email": "john@example.com",
      "delivery_address": {
        "address_1": "14 Highland Close",
        "suburb": "Vredehoek",
        "city": "Cape Town",
        "province": "Western Cape",
        "postal_code": "8001",
        "country": "South Africa"
      },
      "drop_off_instructions": "Leave at front door"
    },
    "parcels": [
      {
        "reference": "PKG-001",
        "description": "Electronics",
        "dimensions": {
          "length_cm": 30,
          "width_cm": 20,
          "height_cm": 15
        },
        "weight_kg": 2.5
      }
    ],
    "carrier_service": "dpd-onx",
    "purchase_label": false,
    "collection_date": "2026-04-10",
    "collection_window": "anytime",
    "label_format": "10_10"
  }'
```

The response includes a `shipment_id`. Hold onto it — you'll need it for subsequent steps.

<Info>
  **Want to skip steps 4 and 5?** Set `purchase_label` to `true` and the shipment will be created, purchased, and the label returned all in one call. The response will include `waybill_number`, `tracking_link`, and `labels` with download URLs.
</Info>

</Step>

<Step title="Purchase the shipment">

<Note>
  This step is only required if you created the shipment with `purchase_label: false` in step 3. If you set it to `true`, the shipment is already purchased and you can skip to step 6.
</Note>

When you're ready to commit, purchase the shipment. This assigns a carrier, generates a waybill number, and creates the shipping label.

```bash
curl -X POST https://api.evership.app/v1/shipments/purchase \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY" \
  -d '{
    "shipment_id": "SHIPMENT_ID",
    "carrier_service": "dpd-onx",
    "collection_date": "2026-04-10",
    "collection_window": "anytime",
    "label_format": ["10_10"]
  }'
```

The response confirms the purchase and returns the assigned waybill number, tracking links, and billing details. Once purchased, the shipment can no longer be edited or deleted — only cancelled.

</Step>

<Step title="Retrieve the label">

<Note>
  The label is returned in the response when creating (with `purchase_label: true`) or purchasing a shipment. This endpoint is only needed if you want to retrieve it again later, or in a different format.
</Note>

Download the shipping label so you can print it and attach it to the parcel.

```bash
curl -G https://api.evership.app/v1/shipments/label \
  -H "apiKey: YOUR_API_KEY" \
  --data-urlencode "shipment_id=SHIPMENT_ID" \
  --data-urlencode "label_formats=10_10"
```

The response contains label URLs for each requested format. Print the label and attach it to the parcel before the carrier arrives for collection.

<Info>
  Available label formats: `10_10` (10 x 10 cm), `10_15` (10 x 15 cm), and `a4_page`. You can request multiple formats in one call by comma-separating them: `label_formats=10_10,10_15,a4_page`.
</Info>

</Step>

<Step title="Track the shipment">

Poll the tracking endpoint to check the current status of your shipment at any time.

```bash
curl -G https://api.evership.app/v1/shipments/tracking-status \
  -H "apiKey: YOUR_API_KEY" \
  --data-urlencode "shipment_ids=SHIPMENT_ID"
```

The response includes the current status (e.g. `collection_requested`, `in_transit`, `delivered`) along with timestamps. See the [Shipment Lifecycle](/shipment-lifecycle) page for a full breakdown of every status.

<Info>
  For real-time updates without polling, set up a [webhook](/api-reference/webhooks/create-webhook) to receive status change notifications as they happen.
</Info>

</Step>

</Steps>
